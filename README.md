# Сайт компании ЮВЕНТА — образовательное оборудование для школ и детских садов

## О проекте

Компания **ЮВЕНТА** поставляет оборудование (интерактивные панели, конструкторы, робототехника, настенные панели) в детские сады и школы. Сайт решает две задачи:
- Привлекает директоров и заведующих полезными ресурсами (ФГОС, ФОП, гранты, инструменты)
- Рекламирует товары компании и генерирует заявки

---

## Технологии

- Чистый HTML / CSS / JavaScript (без фреймворков)
- Адаптивный дизайн (мобильные + десктоп)
- Шрифты: Google Fonts (Roboto) — CDN
- Иконки: Font Awesome — CDN
- Генерация документов: **docx** (библиотека npm, CDN: unpkg.com/docx) — создание .docx прямо в браузере
- Отправка форм: **EmailJS** — бесплатный сервис, не нужен backend (лимит: 200 писем/месяц)
- Аналитика: **Яндекс.Метрика**
- Сохранение состояния чеклистов: **localStorage** (только в пределах одного браузера/устройства)
- Контент (новости, гранты, товары): **JSON-файлы** — редактируются без правки HTML

## Форма заявки — настройка EmailJS

1. Зарегистрироваться на emailjs.com (бесплатно до 200 писем/месяц)
2. Создать Email Service (Gmail или любой SMTP)
3. Создать Email Template
4. Вставить `serviceID`, `templateID`, `publicKey` в `js/form.js`
5. **Обязательно:** в настройках EmailJS → Security → указать допустимые домены
6. Подключить Cloudflare Turnstile для защиты от спама (см. раздел Безопасность)

> **Лимит 200 писем/месяц** может быть исчерпан в первый активный месяц при высоком трафике. При необходимости перейти на платный план EmailJS или заменить на собственный SMTP через серверный бэкенд.

> **Обязательно:** каждая форма, собирающая персональные данные, должна содержать чекбокс:
> ```html
> <label>
>   <input type="checkbox" required>
>   Согласен на <a href="policy.html">обработку персональных данных</a>
> </label>
> ```
> Без этого элемента сбор данных нарушает 152-ФЗ, даже при наличии policy.html.

### Анимации

Все анимации — чистый CSS + JavaScript (Intersection Observer API), без сторонних библиотек.

**1. Hero-анимация на главной странице**
Заголовок, подзаголовок и кнопка плавно появляются при загрузке (fade + slide up):
```css
@keyframes fadeSlideUp {
  from { opacity: 0; transform: translateY(30px); }
  to   { opacity: 1; transform: translateY(0); }
}
.hero-title   { animation: fadeSlideUp 0.6s ease forwards; }
.hero-subtitle { animation: fadeSlideUp 0.6s ease 0.2s forwards; opacity: 0; }
.hero-btn     { animation: fadeSlideUp 0.6s ease 0.4s forwards; opacity: 0; }
```

**2. Появление карточек при скролле**
Intersection Observer отслеживает карточки — они плавно появляются когда входят в область видимости:
```js
// js/nav.js
const observer = new IntersectionObserver((entries) => {
  entries.forEach(e => {
    if (e.isIntersecting) e.target.classList.add('visible')
  })
}, { threshold: 0.1 })

document.querySelectorAll('.card').forEach(c => observer.observe(c))
```
```css
.card { opacity: 0; transform: translateY(20px); transition: opacity 0.5s, transform 0.5s; }
.card.visible { opacity: 1; transform: translateY(0); }
```

**3. Hover-эффекты на карточках**
Карточки приподнимаются и получают тень при наведении:
```css
.card {
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}
.card:hover {
  transform: translateY(-4px);
  box-shadow: 0 8px 24px rgba(0, 0, 0, 0.12);
}
```

> **Важно:** Анимации должны уважать системную настройку `prefers-reduced-motion` — для пользователей с вестибулярными нарушениями:
> ```css
> @media (prefers-reduced-motion: reduce) {
>   *, *::before, *::after { animation: none !important; transition: none !important; }
> }
> ```

## Безопасность

### XSS — межсайтовый скриптинг
**Риск:** высокий. Если вставлять данные из JSON в DOM через `innerHTML` — злоумышленник может внедрить вредоносный скрипт.

**Правило для всего JS-кода:**
```js
// НИКОГДА так:
element.innerHTML = data.title

// ВСЕГДА так:
element.textContent = data.title

// Если нужен HTML (например, ссылка) — только через DOM API:
const a = document.createElement('a')
a.href = data.url        // браузер автоматически экранирует
a.textContent = data.label
container.appendChild(a)
```

### EmailJS — защита публичного ключа
**Риск:** средний. Ключ виден в исходном коде — боты могут слать письма от имени сайта.

**Решение:** в настройках EmailJS → вкладка Security → указать допустимые домены. Тогда ключ сработает только с вашего домена.

### Защита формы от спама
**Риск:** высокий. Без защиты форму заспамят боты за первые дни.

**Решение:** подключить **Cloudflare Turnstile** (бесплатно, без раздражающих картинок):
```html
<script src="https://challenges.cloudflare.com/turnstile/v0/api.js" async defer></script>
<div class="cf-turnstile" data-sitekey="ВАШ_КЛЮЧ"></div>
```

> **Важно:** Turnstile даёт полную защиту только при серверной проверке токена (POST-запрос на `https://challenges.cloudflare.com/turnstile/v0/siteverify`). Без бэкенда токен проверяется только на клиенте — это снижает уровень защиты. Тем не менее Turnstile эффективно блокирует автоматических ботов даже без серверной проверки.

### CDN — проверка целостности файлов (SRI)
**Риск:** средний. Если CDN взломают и подменят файл — вредоносный код выполнится у всех посетителей.

**Решение:** атрибут `integrity` для внешних скриптов и стилей с фиксированными версиями:
```html
<script
  src="https://cdn.example.com/lib.js"
  integrity="sha384-ХЭШФАЙЛА"
  crossorigin="anonymous"></script>
```
Хэш генерируется на [srihash.org](https://www.srihash.org/) для каждой используемой версии библиотеки.

> **Исключения:** SRI нельзя применить к **Яндекс.Метрике** и **Google Fonts** — они обновляют файлы без предупреждения и отдают разные ответы в зависимости от браузера. Добавление `integrity` к этим скриптам сломает их работу.

### Защита от встраивания в iframe (Clickjacking)
**Риск:** низкий. Сайт могут встроить в iframe и обмануть пользователей.

**Решение:** настроить на хостинге HTTP-заголовки (в файле `.htaccess` для Apache или в настройках Nginx/Cloudflare):
```
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
```

### Защита от фишинговых копий сайта
**Риск:** специфичный. Если сайт станет авторитетным, его могут скопировать и разместить вредоносные "документы" под видом ФГОС.

**Меры:**
- Явно указывать официальный адрес сайта на всех страницах
- Зарегистрировать похожие домены (с опечатками) и сделать с них редирект
- Добавить в подвал: "Единственный официальный сайт — [адрес]"
